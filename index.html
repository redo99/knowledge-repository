les design pattern

De création

  • : garantit que l’instance d’une classe n’existe qu’en un seul exemplaire, tout en fournissant un point d’accès global à cette instance.
  • : définit une interface pour créer des objets dans une classe mère, mais délègue le choix des types d’objets à créer aux sous-classes.
  • : permet de créer des familles d’objets apparentés sans préciser leur classe concrète.
  • : permet de construire des objets complexes étape par étape. Il permet de produire différentes variations ou représentations d’un objet en utilisant le même code de construction.
  • : crée de nouveaux objets à partir d’objets existants sans rendre le code dépendant de leur classe.

L'inférence de type

C'est le mécanisme qui permet a un compilateur ou un intérpréteur de recherche le type (le plus général possible) que puisse prendre une expression.

POO

L'Encapsulation

Un principe permettant de garantir l'intégrité des données d'un objet en empêchant l'acces direct a ces données et à l'implémentation de l'objet. La modification des données passe alors par l'utilisation de méthodes appropriées (interfaces).

L'héritage

Specialisation d'une classe à partir d'une classe mère plus générique.

Interdiction de l'héritage
  • Une classe final interdit l'héritage de la dite classe.
  • Une methode final ne peut pas être redéfinit dans une classe dérivée.

Le polymorphisme

  • Ad-hoc (surcharge) : Surcharge d'une methode avec des signatures différentes pour effectuer des traitements différents.
  • Paramétrique : Surcharge d'une methode avec la meme signature mais des paramètres différents, pour réaliser le meme traitement.
  • D'héritage : Redéfinir une methode de la classe mère pour effectuer un traitement different.

L'abstraction

Méthodes abstraites

Méthodes qui permettent de forcer les classes filles a définir (implementer) un comportement (methode) particulier.

Les methodes abstraites, sont définis dans des classes abstraites

Classes abstraites

Une classe abstraite est une classe dont l'implémentation n'est pas encore complete et qui n'est pas instantiable. Elle sert de base a d'autres classes dérivées.

Elle a pour but de s'assurer que les classes filles respecteront le contrat définit pas la classe meme.

Elle définit des attributs, des méthodes pures, et des méthodes abstraites.

Interfaces

Une interface est un contrat que les classes l'implémentant s'engage a respecter. Elle définit les constantes, et les signatures des méthodes à implementer.

Objets immuables

Un Objet immuable est un objet dont l'état ne peux être changé. L'exemple typique est celui de la classe String en Java, qui propose des objets immuables. pour avoir un objet de type chaine de caractères mutable, il faut utiliser StringBuffer.

Contrôle d'accès

  • public : visible pour tous et par conséquent le moins restrictif
  • protected (protégé) : visible pour le package et l'ensemble de ses sous-classes
  • (par defaut): visible uniquement par le package dans lequel il se trouve
  • private (privé) : accessible uniquement dans le contexte dans lequel les variables sont définies (à l'intérieur de la classe dans laquelle il est situé)

III- Gestion de projet

Cycle en V vs. méthodes Agile

  • Le cycle en V est une methode de gestion de projets, basée sur 9 phases successives, alors que le Scrum met en avant la realisation d'un produit d'une manière itérative incrémentale.
  • La livraison en cycle V, s'effectue une fois toutes les fonctionnalités développées (ce qui peut creer un effet tunnel) or en scrum les besoins sont priorisés développés et livrés plus fréquemment. c'est le principe de livrer petit et souvent.
  • En cycle en V, l'étude initiale du besoin est figée toute evolution de ce besoin est considérée comme un avenant, alors que les méthodes agile font preuve d'une souplesse quant a l'évolution des besoins en cours de realisation du produit.
